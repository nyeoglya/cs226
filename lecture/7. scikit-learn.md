### 7-2. Preprocessing/Fitting/GridSearch
- 목적: raw data -> ML-ready matrices (X,y) 만들기
- transformer: fit/transform/fit_transform
	- column transformer
- estimator: fit
- 데이터 분할을 먼저 하고, training set에 대해서만 fit한 다음에, transform을 test set에 쓴다.
- pipeline: impute -> scale/encode -> model
- split 종류: stratified split, groupkfold, kfold/shufflesplit
- grouping: 데이터셋에서 같은 그룹(사람 얼굴 데이터에서 'Tom'이라는 사람의 얼굴 사진)은 train/test에 쪼개지면 안 됨. (전부 다 train에만 들어가거나 전부 다 test에만 들어가야 함)
- categorical encoding: one-hot vector.
- text vectorization: document $d$ -> matrix. BoW(Bag of Words), TF-IDF
- imputation: 부분부분 비어있는 데이터들을 통계적으로 채운다. training set에만 사용.
- standardization: normalization 같은거임. kNN, PCA 등에 사용됨.
	- dimension reduction: PCA, KBest
- ColumnTransformer: 위에서 설명했던 pipeline 각 단계를 전부 조합한다. 각 데이터 종류에 따라 달라짐.
	- numeric: impute -> scale
	- categorial: one-hot
	- text: TF-IDF
- estimator: kNN, Decision Tree(CART)
- pipeline: transformer, estimator step들. 각 transformer는 fit/transformer를 수행하고, 마지막 단계는 estimator여야 한다.

- parameter selection: 각 파라미터를 선택하는 단계.
- f1_macro: imbalance class에 유용함.
