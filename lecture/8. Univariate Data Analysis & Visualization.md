### 8-0. Content
- histogram: bin을 tune한다.
- ECDF: bin-free.
- KDE(kernel density estimation): smooth shape & modes(=peaks). bandwidth 정하는게 가장 중요. bound된 데이터는 transform해서 bias를 피해야 한다.
- Box plot: compact data comparison.
	- Boxen: 큰 크기의 데이터. shows more detail in the tails.
- Rug/Strip, overlay on Box/Violin for validation
- Q-Q Plot
- Log or related transform: heavy right-tail/nonnegative data에 유용. plot이라는 약간 무관함
- scikit-learn 라이브러리에서 제공하는 데이터를 사용한다.
	- Iris, Wine, Diabetes
### 8-1. Histogram, ECDF
- histogram: $\{b_0,\cdots,b_K\}$ 까지의 k개 bin이 있다고 하자.
-  count와 density를 정의할 수 있다.
- freedman-diaconis rule: 더 나은 bin을 결정하는 방법. 기존의 bin count는 noise를 만든다.
- probability vs density: prob.는 높이의 합이 1이 되게 하는 방법/density는 area가 1이 되게 하는 방법
- multivariate하게 사용할 수도 있다.
	- shared bins: 하나의 bin 경계를 pool된 데이터에서 결정하고 그걸 나머지 모든 group에 적용.
	- density normalization: group에 대응하는 밀도. 각 group의 히스토그램의 넓이가 1이 되도록 한다.
- cumulative histogram: ECDF와의 연결점. 점점 늘어나게 하면서 그리는 방법.
	- quantiles, stochastic dominance에 사용한다.
	- ECDF(empirical cumulative density function): $\hat{F}(x)=\frac{1}{n}\sum_{i=1}^n \mathbb{1}\{x_i \le x\}$. 걍 quantiles임.
### 8-2. KDE
- kernel density estimation
- 작은 sample에서 전체 population의 분포를 추정하는 방법.
- local evidence: 각 관측된 $x_i$가 그 자체만을 의미하는 것이 아니라 주변 nbhd를 동시에 의미하는 거임.
- kernel: bell-shaped prob. curve
- 데이터 여러 개를 kernel의 합으로 정의한다.
$$
f_n(x) = \frac{1}{nh}\sum_{i=1}^n K \left( \frac{x-x_i}{h} \right)
$$
- K: kernel function (보통 gaussian임)
- h: bandwidth. 각 kernel의 widthf를 조절하는 값. 이거만 잘 조절하면 됨.
	- 너무 작으면 undersmooth이고, 너무 크면 oversmooth이다.
- 데이터가 bounded라고 해도 여전히 tail을 제공한다.
- KDE는 non-parametric이다. 즉, univariable/multivariable에 무관하다.
### 8-3. Robust Summaries
- median: 50% percentile
- IQR(interquartile range): 75% percentile ~ 25% percentile (기억하기)
- MAD(median absolute deviation): $\text{median}(|x_i-\text{median}(x)|)$
- box: robust summary. quantile summary만 제공한다. 빠르게 읽을 수 있음.
	- median, IQR 범위를 표기하며, min/max($Q_1 - 1.5*IQR$, $Q_3 + 1.5*IQR$)
- violin: box + KDE shape. multimodality
	- bandwidth에 sensitive하다.
	- KDE plot의 장단점을 전부 가지고 있음.
- boxen: 큰 데이터의 tail을 잘 plot하기 위함.
	- nested box을 그린다.
		- 첫번째 box는 25%~75%
		- 두번째 box는 12.5%~87.5%
		- 세번째 box는 6.25%~93.75%
- 작은 데이터 사이즈에서는 box plot. raw data에는 stripplot을 쓴다.
- shape가 중요하면 violin plot을 쓴다.
- 큰 데이터 사이즈에서는 boxen을 고려한다.
- overplotting 문제 발생: jittered point를 추가한다. (노이즈를 추가해서 의도적으로 한 점에 집중된 데이터를 흩뿌려놓음)
### 8-4. Rug, Strip
- rug: one tick per observation
- strip: scatter plot. jitter로 overlap을 막는다.
### 8-5. Q-Q Plot
- quantile-quantile plot: $(F^{-1}(p_i), x_i)$를 그린다. 여기서 F는 reference CDF이다. (보통은 gaussian임)
- reference plot이랑 실제 데이터가 얼마나 비슷한지 체크할 수 있다. 만약 완벽히 따르면 그냥 직선임.
- 데이터가 normal 분포를 얼마나 따르는지 확인할 수 있음.
