### 15-1. Data Model
- structured data model: schema가 독립적으로 존재. 모든 tuple이 이 schema를 따라야 함.
	- relational (data) model: 우리가 지금까지 계속 다루고 있었던 바로 그 모델 = a lot of tables
	- (relational) algebra: set of api, set of operators for the model. manipulation layer.
	- query language: 더 쉽게 algebra를 사용하기 위한 언어.
		- (query) optimizer: query를 algebra로 변환하는 대상.
- SQL: structure query language for relational model. flat model. (no nesting allowed)
- semi-structured data model: data 자체가 scheme을 갖는다.
	- 데이터에 schema 정보가 임베딩되어있음.
	- JSON: kind of nested model
	- XML
### 15-2. Relational Data Model
- ex)
	- Sailors S(sid, sname, rating, age)
	- Boats B(bid, bname, color)
	- Reserves R(sid, bid, day)
	- 여기서 R에 사용되는 primary key는 (sid, bid, day) 전체이다. key는 set of attributes이다.
- 기본적인 SQL query는 다음의 SELECT-FROM-WHERE 형태를 따른다.
`SELECT [DISTINCT] target-list`
`FROM relation-list`
`WHERE qualification`
- relation-list: relation 이름의 리스트. range variables이 포함될 수 있음.
- target-list: 이름 리스트에서 가져온 attribute들.
- qualification: 비교구문.
	- A op c or A1 op A2 같은 거.
	- op로 가능한 건 =, <, >, <=, >=, /=가 있다.
	- AND, OR, NOT을 사용가능하다.
- DISTINCT: 중복 제거. 기본적으로는 중복을 제거하지 않는다.
- like: pattern matching
- set operator: union, intersect
	- 주어진 relation R과 S에 set operator를 사용하기 위해서는, R와 S는 union-compatible(schema-compatible보다 더 완화된 개념)해야 한다.
	- union-compatible: attribute 개수가 같아야 함. 각 attribute의 type도 같아야 함. (이름은 상관이 없음)
	- 최종 결과는 dbms에서 구현하는 방식에 따라 다르다.
	- set operation은 연산이 끝나고 자동으로 duplicate를 전부 다 지운다.
		- duplicate 남기고 싶다면, union 대신에 union with/intersect 대신에 intersect with를 써야 한다.
- nested queries: `SELECT S.sname FROM Sailors S WHERE f(S)`
	- `f(S) = S.sid IN SELECT R.sid FROM Reserves R WHERE R.bid = 103`
	- optimizer가 join query로 변환한다. 근데, 그건 내 알빠 아님.
- any: existential quantifier
- nested predicates
	- type N: no join with outer. no aggregate
	- type A: no join with outer. inner SELECT is an aggregate
	- type J: join with outer in inner WHERE. no aggregate
	- type JA: join with outer in inner WHERE. inner SELECT is an aggregate
- type JA에서 all red boat만을 대상으로 하는 경우도 만들어보기
- exist(Q): check if Q has at least one tuple
- unique(Q): check if there is no duplicate in results of Q
	- unique(empty): 설명에 의하면 이것도 참이다. ==한번 체크해보기==
